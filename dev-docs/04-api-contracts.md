# 04 — API Contracts (Frontend ↔ Backend)

This document is the **single normative contract** between the React frontend and the Go backend. Implementations live in `internal/services/`; DTO structs in `internal/models/`; TS types are generated by `wails3 generate bindings` — never hand-written.

Conventions:
- All methods take `context.Context` (omitted below for brevity in tables).
- All methods can return `AppError` (§8).
- Streams are not returned as channels to the frontend; they are **started** by a method and delivered via events (§7) keyed by a session/stream ID.
- Mutations with risk ≥ `needs_confirmation` use the two-step plan/apply pattern (§5).

## 5a. RegistryService (accounts, login, publish support)

Manages registry authentication on the **backend OS** by delegating to the standard Docker credential flow (`docker login`), so credentials live in Docker's credential store/helper — never in Cairn's DB ([05-security.md §4]).

```go
ListRegistryAccounts() ([]RegistryAccount, error)
// RegistryAccount{Registry, Username, Source(credHelper|credsStore|authsFile), LoggedIn bool, LastVerifiedAt}
// Parsed from backend ~/.docker/config.json + helper listing; passwords never read back.
Login(req RegistryLoginRequest) error
// RegistryLoginRequest{Registry string /* default docker.io */, Username, Secret string,
//                      SecretKind: password|access_token}
// Executes `docker login <registry> -u <user> --password-stdin` via provider; secret piped,
// never an argv element, never persisted by Cairn, redacted everywhere.
Logout(registry string) error                       // docker logout <registry>
TestAuth(registry string) (*RegistryAuthStatus, error)  // verifies token grant works
KnownRegistries() ([]RegistryPreset, error)
// presets: Docker Hub (docker.io), GitHub (ghcr.io), GitLab (registry.gitlab.com),
// Quay (quay.io), Google Artifact Registry, custom…
```

Notes: Docker Hub logins with 2FA require an access token (UI explains this, links to hub token page). Push authorization errors surface as `E_REGISTRY_AUTH` with the failing registry named. Update checks (modules/08) automatically benefit from logged-in accounts — private images stop showing `auth_required`.

## 7. Events (backend → frontend)

All events carry `{ts, providerID}` plus payload. Emitted via Wails v3 events; names are stable API.

| Event | Payload | Notes |
|---|---|---|
| `app:ready` | bootstrap state | once at startup |
| `provider:changed` | ProviderSummary | active provider switched |
| `provider:status` | ProviderStatus | health change |
| `provider:install:progress` | {step, totalSteps, message, done, error} | |
| `docker:connected` / `docker:disconnected` | {host, context} / {reason} | drives global banner |
| `objects:changed` | {kind: container\|image\|volume\|network, ids[]} | coalesced ≤ 250 ms from Docker events |
| `project:changed` | {projectID} | re-fetch hint |
| `stats:sample` | {streamID, samples[]} | coalesced to latest per container |
| `logs:lines` | {streamID, lines []LogLine{ts,source,level?,text}} | batched ≤ 50 ms / 200 lines |
| `logs:eof` / `logs:error` | {streamID, error?} | |
| `terminal:data` | {sessionID, dataBase64} | |
| `terminal:closed` | {sessionID, exitCode} | |
| `image:pull:progress` | {streamID, layerID, status, current, total} | |
| `image:push:progress` | {streamID, layerID, status, current, total} | same shape as pull |
| `updates:check:progress` | {jobID, done, total, current} | |
| `updates:applied` | UpdateHistoryItem | success or failure |
| `job:progress` | {jobID, phase, message, pct?} | generic (backup, install, update apply) |
| `job:done` | {jobID, result, error?} | |
| `notification` | {level, title, body, topic} | feeds notification center |

## 8. Error codes (`AppError.Code`)

```text
E_DOCKER_UNREACHABLE      daemon ping failed → banner + repair hints
E_PROVIDER_NOT_READY      provider not installed/running → onboarding/repair
E_PROVIDER_DETECT_FAILED
E_COMPOSE_NOT_FOUND       compose plugin missing
E_COMPOSE_INVALID         compose config invalid (detail = validation output)
E_WORKDIR_MISSING         project working dir not found (moved/deleted)
E_PERMISSION_DENIED       docker socket permission → show permission options
E_REGISTRY_AUTH           registry auth required
E_REGISTRY_RATE_LIMIT
E_REGISTRY_UNREACHABLE
E_NOT_FOUND               object disappeared (stale cache)
E_CONFLICT                e.g. remove running container without force
E_PLAN_EXPIRED            plan/apply two-step timed out (plans expire in 10 min)
E_CONFIRMATION_REQUIRED   apply called without satisfying typed-name confirmation
E_TIMEOUT
E_CANCELLED
E_INTERNAL
```

Frontend mapping rules: `E_DOCKER_UNREACHABLE|E_PROVIDER_NOT_READY` → global state banner + redirect affordance; `E_PERMISSION_DENIED` → permission dialog ([ui/02-onboarding.md §6]); registry errors → per-row status, never modal spam; `E_NOT_FOUND` → silent refetch + toast.
